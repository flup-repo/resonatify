## 11. Keyboard Navigation Map

**Global Shortcuts:**
- `Cmd+N` - New Schedule
- `Cmd+,` - Settings
- `Cmd+0` - Show/Hide Window
- `Cmd+Q` - Quit Application
- `Cmd+W` - Close Window (hide to menu bar)

**Within Main Window:**
- `Tab` - Navigate between interactive elements
- `Space` - Toggle checkboxes/switches
- `Enter` - Activate focused button
- `â†‘` / `â†“` - Navigate schedule cards (future)
- `Delete` - Delete selected schedule (with confirmation)

**Within Modal:**
- `Tab` / `Shift+Tab` - Navigate form fields
- `Enter` - Submit form (if on button or last field)
- `Escape` - Cancel and close modal
- `â†‘` / `â†“` - Adjust slider/dropdown values

**Focus Order (Main Window):**
1. "+ New Schedule" button
2. Settings button
3. Refresh button
4. First schedule card (toggle)
5. First schedule card (Test button)
6. First schedule card (Edit button)
7. First schedule card (Delete button)
8. Second schedule card (toggle)
9. ... (continues for all cards)

**Focus Indicators:**
- Blue outline (2px) around focused element
- High contrast for accessibility
- Visible in both light and dark themes

## 12. Accessibility Annotations

**Screen Reader Labels:**
- All icon buttons have aria-labels
- Schedule cards have descriptive labels (e.g., "Morning Meditation schedule, 7:00 AM daily, enabled")
- Form inputs have associated labels
- Status messages use aria-live regions

**Color Contrast:**
- WCAG AA minimum (4.5:1 for body text)
- WCAG AAA preferred (7:1 for body text)
- Interactive elements: 3:1 minimum

**Touch Targets:**
- Minimum 44x44px (Apple HIG)
- Spacing between targets: 8px minimum
- Forgiving click areas (extend beyond visual bounds)

**Focus Management:**
- Auto-focus first input in modals
- Return focus on modal close
- Skip navigation links (if complex UI)
- Logical tab order
